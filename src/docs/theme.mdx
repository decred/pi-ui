# Theming

## Create a custom theme

The recommended way to create a custom theme is to create a file named `theme.js`, you can add your custom values and override existing ones.

```js
// theme.js

export default {
  "color-primary": "red",
  "btn-color": "var(--color-primary)"
};
```

## Use your theme in your application

Pi-ui exposes an `useTheme` hook that accepts a `theme` argument. All CSS properties will be updated accordingly.


```js
import React from "react";
import { render } from "react-dom";
import { useTheme } from "pi-ui";
import theme from "./theme";

const App = () => {
  useTheme(theme);
  return (
    // App jsx
  )
};

render(<App />, document.getElementById("root"));
```

## Default theme

There are two default themes, `defaultLightTheme` and `defaultDarkTheme`. They are both name exported from pi-ui.

```js
import { defaultLightTheme, defaultDarkTheme } from "pi-ui";
```

## Use theme property

You can use any property from a theme in your code:

```js
import React from "react";
import { useTheme, defaultLightTheme } from "pi-ui";

const Test = () => {
  const theme = useTheme(defaultLightTheme);
  /*
  theme = {
    "color-primary": "#2970ff",
    "color-primary-dark": "#091440",
    "color-primary-light": "#2ed8a3"
    ...
  }
  */
}
```

## Add font assets

You can provide a second parameter to the `useTheme` hook to add font assets

```js
import React from "react";
import { defaultLightTheme, useTheme } from "pi-ui";

import TomorrowLight from "../assets/fonts/tomorrow/Tomorrow-Light.ttf";
import TomorrowRegular from "../assets/fonts/tomorrow/Tomorrow-Regular.ttf";
import TomorrowSemiBold from "../assets/fonts/tomorrow/Tomorrow-SemiBold.ttf";

const fontConfig = {
  fontFamilyText: "Tomorrow",
  regularUrl: TomorrowRegular,
  semiBoldUrl: TomorrowSemiBold,
  lightUrl: TomorrowLight,
  format: "truetype" // .ttf assets => 'truetype'
};

const App = () => {
  defaultLightTheme["font-family-text"] = "Tomorrow";
  useTheme(defaultLightTheme, fontConfig);
  return (....);
};
```

* Note that you **have** to pass a theme to `useTheme` everytime if you are using a custom theme, otherwise it will be set to the `defaultLightTheme`.
